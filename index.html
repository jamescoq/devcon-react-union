react-union could be the solution

What is react-union?

What is react-union?

  • solution, finds apps on a page
  • call them as widgets, refer as such, from now
  • combine widget into a single app
  • forming single virtual DOM
  • solution to all troubles

How can react-union help?

single virtual DOM

shared app state
shared react context
  • thanks to SVD - able to share data
  • redux state, context
  • initial data, theming, localization

simple integration with Liferay

initial data from Liferay
  • integration with LF is easy
  • using LF AMD loader, but theme could be used as well
  • provides initial data

webpack configuration

HMR
proxy
  • another feture - preconfigured webpack
  • develop over dev server without LF
  • develop over running LF instacne with HMR
  • code splitting

High level architecture

  • essential part - Union comp.
  • Root of react app which encapsulates widgets
  • on LF - define DOM marks and widgets descriptors
  • include loading of Union component
  • after init it scans current page and looks for widget descriptors
  • initializes widgets on proper place using react portals under the hood
  • preconfigured webpack which compile into several JS files
  • can be loaded by AMD loader

Union component


                        <Union routes={routes} />
  • one required property called routes


    const routes = [
        {
            path:'maps',
            getComponent(done) {
                import('../widgets/Maps').then(done);
            },
        },
        {
            path:'search',
            getComponent(done) {
                import('../widgets/Search').then(done);
            },
        },
    ]
  • an array of objects describing implemented widgets
  • and a render path
  • path match - widget descriptor


    import { justRender } from 'react-union';

    justRender(<Union routes={routes} />);
  • In order to render the Union component, we provide justRender
  • replaces render method from ReactDOM
  • adding dynamically a real DOM tag into whicih Union component is rendered 


    const Root = () => (

  • decorate the Union component

withWidgetContext



<script data-union-widget="myCoolWidget" data-union-container="myCoolWidget"
type="application/json">
    {
        "home": "www.example.com",
        "api": "/api"
    }
</script>
                        

import { withWidgetContext } from 'react-union';

const NestedComponent = ({ namespace, data: { home, api } }) => (
    <div>
       ...
    </div>
);
export default withWidgetContext(NestedComponent);
  • upper part a widget descriptor, JSON as initial data
  • available as props in Widget's Root component
  • but not in nested components
  • so the purpose of withWidgetContext is to make the data available in nested components

pass common data
